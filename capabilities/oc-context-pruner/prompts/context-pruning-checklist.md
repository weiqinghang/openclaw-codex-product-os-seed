# 上下文裁剪清单

在把任务发给 `Codex` 前，`OpenClaw` 必须先完成以下检查。

## 1. 先定单一目标

- 用一句话描述这次唯一目标
- 如果一句话里出现两个动词目标，先拆开，不要直接下发
- 如果当前还在讨论值不值得做，停在 `OpenClaw`

## 2. 标明当前阶段

阶段只允许是以下之一：

- 项目收敛
- 切片定义
- 技术方案
- 实现
- 验证
- 发布准备

无法判断阶段时，不下发。

## 3. 只保留最小必要输入

优先保留：

- 当前 `brief / spec / plan / tasks`
- 当前代码目录或文件范围
- 当前设计链接
- 当前验证记录

默认删除：

- 无关 feature 历史
- 未来版本想法
- 长篇会议纪要
- 与当前切片无关的设计讨论

## 4. 缺关键输入就停止

出现以下任一情况，返回 `OpenClaw`，归类为“需补充输入”：

- 验收标准缺失
- 代码范围不明
- 设计定版缺失但任务依赖设计
- 当前目标仍然模糊

## 5. 输出格式

输出必须落入标准任务包，并满足：

- 只有一个当前目标
- `允许使用` 明确
- `不要使用` 明确
- `预期产出` 可验证
- `升级条件` 可执行

## 6. 自检

发送前再问三件事：

1. `Codex` 是否只需处理一个切片或一批任务？
2. 当前包里是否出现了“顺便”“另外”“后续再说”这类跨范围内容？
3. 如果删掉历史对话，`Codex` 还能否完成当前目标？

任一答案为“否”，继续裁剪。
