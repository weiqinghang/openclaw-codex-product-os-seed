# 项目规划图

这份目录专门用于维护本仓库的整体规划、已实现情况和未来候选故事。

## 如何使用

- 已实现的主题，写到对应栏目下的“已实现”
- 已经确认但尚未进入当前迭代的主题，写到“待做”
- 头脑风暴产生的新方向，也先落到这里，而不是直接塞进当前迭代
- 只有当某个故事被确认进入执行，才把它下钻到 `docs/current-iteration.md` 或具体 `specs/features/...`

## 使用原则

- 这里维护的是全局规划，不是当前任务清单
- 这里可以同时容纳已实现、进行中、待做、脑暴候选
- 这里优先表达主题、边界和状态，不追求实现细节
