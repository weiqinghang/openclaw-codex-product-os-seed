# 语言规则

## 目标

在种子项目初期，只维护一套中文内容，避免中英文并行时的疏漏和漂移。

## 规则

### 1. 路径和命名使用英文

以下内容继续保持英文：

- 路径
- 文件名
- 目录名
- slug
- 变量名
- 命令
- 脚本名

### 2. 说明性内容统一中文

当前阶段，所有说明性内容统一使用中文：

- README
- `docs/`
- `specs/`
- `templates/`
- GitHub issue / PR 模板中的展示文字

### 3. 真实工件只保留一套

真实执行工件只使用一套标准文件名：

- `brief.md`
- `spec.md`
- `plan.md`
- `tasks.md`

### 4. 英文版本延后

等流程、规则、样例稳定后，再统一翻译一套英文版本，而不是在当前阶段同步维护双语。

### 5. 新术语必须入表

当文档、模板、示例、issue 模板或 PR 模板中出现新的专有术语时，必须同步更新 [术语表](./glossary.md)。
