# Contributing

## Goal

Contributions should improve the seed in ways that are reusable across multiple downstream projects.

## Good changes

- Clearer workflow definitions
- Better templates
- Small bootstrap scripts
- CI checks that enforce process quality
- Documentation that reduces ambiguity for humans and agents

## Avoid

- Project-specific business logic
- Framework-specific assumptions unless clearly isolated
- Large unrelated edits in one change

## Change rules

1. Update docs when behavior or workflow changes.
2. Update templates when the constitution or lifecycle changes.
3. Prefer small reviewable commits.
4. Include verification notes in pull requests.

## Suggested pull request shape

- Why this change is needed
- What part of the operating system changed
- How it was verified
- What downstream projects may need to update
