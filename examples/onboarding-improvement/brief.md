# Feature Brief

## Metadata

- Title: Improve first-session onboarding guidance
- Requested by: Product
- Date: 2026-03-07
- Priority: High
- Desired timeline: 1 week

## 1. Request Summary

新用户首次创建工作区后，不知道下一步该做什么，导致很多人在 1 分钟内离开。需要增加一段轻量引导，帮助用户完成首次关键动作。

## 2. User and Context

- Primary user: First-time workspace creator
- Triggering scenario: User creates a workspace and lands on an empty dashboard
- Current workaround or failure: 用户只能自行摸索，或退出等待后续再看

## 3. Desired Outcome

让首次进入空白工作区的用户能在 30 秒内理解下一步动作，并完成至少一次关键操作。

## 4. Constraints

- Time: 1 week
- Technical: 不能重构整个 dashboard
- Compliance/security: 无额外要求
- Dependencies: 需要产品确认 CTA 文案

## 5. Evidence

- User feedback: 多位测试用户表示“进来后不知道先做什么”
- Data points: 空白页进入后的首分钟流失偏高
- Stakeholder input: 产品希望先验证轻量引导是否有效

## 6. Initial Slice Proposal

先在空白 dashboard 增加一张 onboarding 卡片，包含一句说明、一条主 CTA、一个次级帮助链接。

## 7. Open Questions

- CTA 应直接进入创建第一个任务，还是进入引导页？
- 卡片何时消失？
