# 功能规格

## 元数据

- 标题：空白页引导卡片
- 状态：草稿
- 负责人：产品 + 工程
- 日期：2026-03-07
- 相关请求：slice-1-card

## 1. 问题

空白工作区用户进入 dashboard 后缺少明确的第一步动作提示，导致用户难以开始。

## 2. 用户与场景

受影响用户是第一次进入空白工作区的新用户。问题发生在创建工作区后首次进入空白 dashboard 的场景。

## 3. 期望结果

用户能够理解建议动作，并通过主按钮进入第一步关键行为。

## 4. 范围

### 本次范围

- 在空白 dashboard 上显示一张引导卡片
- 展示一句说明文案
- 展示一个主动作按钮

### 本次不做

- 不记录用户是否已完成首次动作
- 不做跨会话隐藏
- 不做次级帮助入口

## 5. 用户场景

1. 作为首次进入空白工作区的用户
2. 我希望马上知道建议的第一步动作
3. 以便我可以快速开始使用产品

失败路径：

- 用户不点击按钮，页面仍保持可用

## 6. 功能性要求

- 仅在空白 dashboard 场景展示卡片
- 卡片包含标题、描述和一个主动作按钮
- 点击按钮后进入第一步关键动作页面

## 7. 非功能性要求

- 性能：不显著增加首屏延迟
- 可靠性：主按钮失败时有清晰反馈
- 可访问性：按钮可被键盘聚焦
- 可观测性：记录卡片曝光和主按钮点击

## 8. 验收标准

- [ ] 空白工作区用户进入 dashboard 时可看到卡片
- [ ] 非空工作区用户看不到卡片
- [ ] 点击主按钮后可进入预期页面
- [ ] 曝光和点击事件能够被记录

## 9. 待确认问题

- 主按钮文案是否会影响点击率？

## 10. 风险与假设

- 假设：存在可靠的空白工作区判断逻辑
- 风险：如果按钮文案不够清楚，可能看得到但不点击

## 11. 备注

这是验证性切片，不承诺完整 onboarding 体验。
