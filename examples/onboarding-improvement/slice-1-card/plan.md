# 实现计划

## 元数据

- 功能：空白页引导卡片
- 负责人：工程

- 规格链接：`examples/onboarding-improvement/slice-1-card/spec.md`

## 1. 摘要

在 dashboard 空状态区域新增一张轻量引导卡片。通过已有空白工作区判断逻辑控制展示，主按钮直达第一个关键动作入口，并记录曝光和点击事件。本切片不处理隐藏与持久化。

## 2. 当前状态

当前空白 dashboard 只有基础说明，没有明确建议动作。

## 3. 方案设计

### 受影响组件

- Dashboard 空状态组件
- 路由入口
- 埋点事件记录

### 数据模型变更

- 无

### 接口或契约变更

- 无

### 界面或流程变更

- 空白页新增一张引导卡片

## 4. 备选方案

- 方案 A：直接做完整 onboarding
- 方案 B：先只做引导卡片

选择 B，因为更小、更快、更适合验证。

## 5. 交付策略

### 纵向切片

1. 接入卡片 UI
2. 接入主按钮跳转
3. 接入曝光与点击埋点

### 发布方式

- 先直接对空白工作区生效

### 回滚方式

- 通过开关关闭卡片显示

## 6. 测试策略

- 单元测试：空白判断与展示条件
- 集成测试：按钮跳转和事件记录
- 手动验证：空白 / 非空工作区的展示差异

## 7. 风险

- 技术风险：空白判断逻辑可能误判
- 产品风险：按钮文案可能不够清楚
- 运行风险：埋点缺失会削弱验证价值
