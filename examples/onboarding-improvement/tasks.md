# 任务拆解

## 元数据

- 功能：空白 dashboard 引导卡片
- 计划链接：`examples/onboarding-improvement/plan.md`
- 负责人：工程

## 任务列表

### 任务 1

- 目标：在空白 dashboard 展示 onboarding 卡片
- 输入：`spec.md`、dashboard 空状态组件
- 变更：新增卡片 UI 和空白判断逻辑
- 验证：手动确认空白 workspace 可见、非空 workspace 不可见
- 完成定义：UI 已接入且展示条件正确

### 任务 2

- 目标：接入主 CTA 跳转与曝光、点击埋点
- 输入：路由、埋点 hooks
- 变更：绑定 CTA 行为与埋点
- 验证：手动点击可跳转；事件日志中可看到曝光和点击
- 完成定义：CTA 和关键埋点可用

### 任务 3

- 目标：完成首次关键动作后隐藏卡片
- 输入：关键动作完成状态
- 变更：增加隐藏条件和回归测试
- 验证：完成首次关键动作后重新进入 dashboard，卡片不再显示
- 完成定义：隐藏逻辑可靠通过测试

## 依赖说明

- 任务 3 依赖关键动作完成状态可判断

## 执行规则

- 保持任务可以独立审查
- 优先做薄而完整的纵切
- 每个任务都绑定验证方式
