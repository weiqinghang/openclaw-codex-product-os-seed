## 综合评价

### 评价方法

每轮按 6 个检查项计分：

1. 请求分类符合预期
2. 读取工件符合最小必要集
3. 四段式输出完整
4. 边界合规（无越权）
5. 升级路径正确
6. 应回写工件已回写

综合通过率 = 符合预期项数量 / 总检查项数量

说明：

- 这是**综合评价**，用于看当前稳定度
- 它**不替代** `docs/current-iteration.md` 中 `v0` 的硬验收门槛
- CLI 验证可计入综合评价，但不能替代“连续 3 轮真实飞书请求”
