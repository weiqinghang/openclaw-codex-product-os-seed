# 验收期线程补充

## 元数据

- 功能：飞书直连的 seed 项目维护 Agent
- 场景：本轮 seed 维护 feature 的验收期线程补充
- 长期规则入口：Agent 层 `THREAD-WORKFLOW.md`

## 1. 定位

本文件不再定义 `龟丞相` 的长期线程工作流。

本文件只负责：

- 本轮验收期异常场景与升级动作
- 稳定性验证要求
- 与 `validation-log.md` 的关系
- 与 `docs/current-iteration.md` 的状态回写关系

长期线程行为、默认读取顺序、默认推进规则、默认汇报格式，统一由 Agent 层 `THREAD-WORKFLOW.md` 负责。

## 2. 验收期要求

验收期内，`龟丞相` 的线程输出需要满足：

1. 遵守长期线程准则，不偏离默认读取顺序与边界
2. 每轮结果都可回填到 `validation-log.md`
3. 每轮判断都能支撑 `docs/current-iteration.md` 的稳定性验收结论
4. 出现异常场景时，按本文件定义的升级动作处理

## 3. 与验证台账的关系

每轮真实飞书请求若进入验收视角，至少需要补这些证据：

- 场景与输入
- 请求分类
- 读取工件
- 产出质量
- 是否越权
- 是否正确升级
- 结果判定
- 风险与修正动作

记录位置：

- `specs/features/openclaw-seed-maintainer-agent/validation-log.md`

## 4. 与状态回写的关系

验收期内，验证结果至少影响这两处：

- `docs/current-iteration.md`
- `docs/00-story-map/index.md`

回写原则：

- 当前轮次新增稳定性口径或验证进展时，回写 `docs/current-iteration.md`
- 验收进展足以影响故事地图说明时，回写 `docs/00-story-map/index.md`
- 在未达到 `done` 口径前，不提前改故事状态结论

## 5. 异常场景与升级动作（验收期）

| 异常场景 | 识别信号 | 处理动作 | 升级目标 |
| --- | --- | --- | --- |
| 分类不稳定 | 同类请求连续两轮归类不一致，导致读取偏差 | 暂停推进，先回报“分类不稳定 + 备选分类 + 影响面” | 人类 |
| 读取范围漂移 | 未按最小必要读取，出现整包注入倾向 | 立即收缩读取集，补充“为何读取这些工件” | 人类（若反复出现） |
| 边界冲突 | 触发跨仓需求、要求改默认职责、要求自动提交/push | 明确拒绝执行越界动作，列待决策项 | 人类 |
| 结构化大改 | 涉及跨文件重组、批量回写、模板体系调整 | 先产出任务包与验收口径，再转执行 | Codex |
| 验收证据不足 | 仅有结论，没有可追溯记录 | 回写 `validation-log.md`，补齐证据链 | 无（必要时人类） |
